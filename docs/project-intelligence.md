## Tech Stack
**Language(s):** TypeScript (ESM, `"type": "module"`)
**Frameworks:** pi coding agent extension SDK, Playwright, Mozilla Readability, jsdom, Turndown + GFM plugin
**Infrastructure:** Local Playwright browser (or remote via CDP/WS); no Docker, no cloud services
**Package Manager:** npm

## Goals
- Provide the pi coding agent with headless browser capabilities (search, fetch, extract, crawl websites)
- Minimum dependencies — only `npx` needed to get started; everything else self-installs
- Zero API keys required — no third-party services to sign up for
- Zero Docker requirement — runs purely on npm packages
- SSRF-safe by default — block private/loopback addresses unless explicitly allowed
- Respect robots.txt and rate-limit during crawls

## Non-Goals
- This is a set of tools, not a research assistant — the agent decides how to use them
- Not a general-purpose browser or scraping framework
- Not a search engine index or caching layer
- No cloud/hosted offering — fully self-hosted local
- No persistent storage or database integration

## Boundaries
- **Entry points:** `index.ts` (pi extension factory), `browser.ts` (fetch engine), `extract.ts` (Readability), `search.ts` (SERP scraping), `crawl.ts` (BFS crawler), `markdown.ts` (HTML→markdown), `config.ts` (env config)
- **External services:** Public search engines (DuckDuckGo, Bing, Brave) scraped via Playwright; arbitrary websites fetched via static/browser fetch
- **Exposed interfaces:** 4 pi tools (`web_fetch`, `web_extract`, `web_search`, `web_crawl`), `/webtools` command, 4 CLI flags, env vars
- **Constraints:** Only http/https URLs, Playwright binary required (or remote CDP/WS), SSRF guard blocks private IPs by default, caps/limits on all fetches and crawls
